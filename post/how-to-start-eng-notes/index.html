How to Start Taking Engineering Notes

Sun, Jan 23, 2022

Anything of serious complexity is too large to exist purely in someone’s head. This becomes especially true during engineering projects with others. Therefore, it’s critical to write down ideas and create abstractions that make such complexity easier to understand and handle.

Here are some tips for taking notes for software engineering.

Tips

Note driven development

Write down tests before you begin. If you have something you’re building, write down tests that you’d like to pass. This can be a simple sentence. It’s kind of like test-driven-development but without all the effort. Even if you’re moving fast you can easily add this so it’s easy to adopt. It’s in that perfect low-effort-high-pay-off zone.

In addition, if you start up a note for tests it’s easy to document small things that happen along the way of actual development. For example, if you do a minor refactor you can write it down to make sure you have a note of it somewhere or to add to your PR later.

Don’t duplicate

Write down things that catch your eye but aren’t part of what you’re doing. It’s easy to get distracted by things and to want to go down the refactor rabbit hole. I have a “debt” note that I use to keep track of these things. Rather than getting side-tracked it helps me stay focused on the task but also lets me remember what I need to do later.

Decision matrices

If you have multiple ideas of how to accomplish something then write down ideas and pros/cons. This will let you keep everything in mind and be able to have a cognizant discussion about which choice is best for the situation. Brag document See Julia Evan’s exceptional piece. Also, keep it going for different quarters so it’s easy to see it broken up.

Tools

  • Diagrams
    • LucidChart - The best I’ve found for making diagrams. The downside is it costs money.
    • Excalidraw - This is simpler than lucid chart but is free. My go to for personal projects. Also, Kent C. Dodds uses it…
    • Ascii diagrams - There are plenty of cool tools online. Please, it’s easy to save since it exports to plain text.
  • Note Taking
    • Google docs - Great for collaboration and sharing.
    • Markdown editor - Markdown is my suggestion for personal notes. It’s easy to share for when you need to share it but also lightweight.
      • Bear - Great experience for tagging
      • Obsidian - If you link to link docs this is the best option out there
      • VS Code - Simple works as well
